# HistQuantOpt 项目审查报告

审查日期：2026-06-17

## 结论

项目主流程清晰，已覆盖行情缓存、CNE6 风险模型、优化、真实回测和 HTML 报告生成。但当前还不适合作为“可信回测 / 可复现包”交付，主要风险集中在停牌执行口径、依赖声明、测试缺失和文档漂移。

## 主要问题

### P1 停牌处理从优化到回测不一致

优化器把 `SUSPENDED` 直接约束为 `w=0`，但 `docs/design.md` 要求停牌股票固定为上期权重。真实回测里，停牌且无有效 VWAP 的卖单会直接跳过，未进入延期卖出队列，导致目标权重和真实持仓系统性偏离。

相关位置：
- `portfolio_optimizer/optimizer/index_enhance.py`
- `portfolio_optimizer/optimizer/alpha_max.py`
- `portfolio_optimizer/backtest/realistic_engine.py`
- `docs/design.md`

建议：停牌股票在优化中固定为 `prev_weight`，并从可交易预算中剥离；真实回测中无法成交的卖单应明确进入延期队列，直到可交易或新目标覆盖。

### P1 依赖声明不可靠

`pyproject.toml` 将 `cvxpy` 放在 optional，但包顶层默认导入优化器，实际 `import portfolio_optimizer` 就依赖 `cvxpy`。报告模块依赖 `plotly`，但 `pyproject.toml` 未声明。`requirements.txt` 又缺少文档要求的 `polars`、`plotly`、`pyyaml`。

相关位置：
- `pyproject.toml`
- `requirements.txt`
- `portfolio_optimizer/__init__.py`
- `portfolio_optimizer/backtest/report.py`

建议：统一依赖来源。若优化器是核心功能，应把 `cvxpy/clarabel/plotly` 纳入基础依赖；若坚持 optional，应避免顶层导入 optional 模块。

### P1 缺少自动化测试

`python3 -m pytest` 收集到 0 个测试。优化约束、涨跌停 / 停牌回测、配置解析、风险模型 as-of 对齐都是高风险逻辑，目前没有回归保护。

建议优先补：
- 优化器约束测试：权重和、单票上限、行业 / 风格约束、停牌固定、涨跌停不可买卖。
- 回测执行测试：T+1 VWAP、涨停买入失败留现金、跌停 / 停牌卖出延期。
- 配置测试：两个 demo config 均能完整跑通到权重输出。
- 数据加载测试：alpha 长表 / 宽表、YYYYMMDD 日期解析。

### P2 `weight_lower` 配置未生效

指数增强配置和文档暴露了 `weight_lower`，代码读取该字段，但优化器只约束了 `w <= weight_upper`，没有实现 `w >= weight_lower`。

相关位置：
- `configs/index_enhance_demo.yaml`
- `portfolio_optimizer/optimizer/index_enhance.py`
- `docs/操作指南.md`

建议：补上 `w >= weight_lower`，或从配置和文档中移除该参数。

### P2 文档承诺硬流动性约束，但实现只有软惩罚代理

`docs/design.md` 写了基于 ADV 的参与率硬约束，但优化器没有使用 `snapshot.adv` / `portfolio_value` 生成硬约束。目前只有 `turnover_penalty` 下的成本向量软惩罚。

相关位置：
- `docs/design.md`
- `portfolio_optimizer/pipeline/universe.py`
- `portfolio_optimizer/optimizer/alpha_max.py`
- `portfolio_optimizer/optimizer/index_enhance.py`

建议：若需要真实流动性控制，增加 `abs(w - prev_weight) * portfolio_value <= participation_rate * adv`；否则文档改成“流动性加权换手惩罚”。

### P2 求解后归一化可能破坏硬约束

两个优化器求解后执行 `clip + normalize`。当解已经贴近上限或约束边界时，归一化可能把股票权重放大到超过 `weight_upper`，也可能破坏行业、风格、换手约束。

相关位置：
- `portfolio_optimizer/optimizer/alpha_max.py`
- `portfolio_optimizer/optimizer/index_enhance.py`

建议：求解后做完整后验约束校验。若违反约束，应返回 infeasible、保留原求解值并标记警告，或用显式投影方法重新修正。

### P2 `backtest_pipeline.run_backtest()` 与当前配置不匹配

`run_backtest()` 读取 `output.nav`、`output.report`、`output.report_title`，但两个 demo config 只有 `output.weights`，直接使用配置运行会报错。

相关位置：
- `portfolio_optimizer/pipeline/backtest_pipeline.py`
- `configs/alpha_max_demo.yaml`
- `configs/index_enhance_demo.yaml`

建议：补齐 config 输出字段，或让 `run_backtest()` 对缺失字段使用默认路径。

### P3 文档漂移

`docs/design.md` 的模块结构仍引用不存在的 `barra_factors.py`、`risk_model.py`、`constraints.py`、`portfolio.py`。`portfolio_optimizer/__init__.py` 指向不存在的 `docs/manual.md`。指数增强方法文档标题写 HS300，但当前默认配置和操作指南是中证1000。

相关位置：
- `docs/design.md`
- `docs/index_enhance_method.md`
- `portfolio_optimizer/__init__.py`

建议：更新设计文档到当前目录结构，并统一“默认指数增强基准”为中证1000或显式说明可切换。

## 处理结果（2026-06-17）

**已修复**
- 依赖声明：`cvxpy/clarabel/plotly` 纳入 `pyproject` 基础依赖；`requirements.txt` 对齐并移除未用的 matplotlib/seaborn。干净环境可 `pip install -e .` 后直接 import/跑 demo。
- `weight_lower`：对全部股票加 `w>=lower` 会逻辑错误（需 MIP 才能表达"持有则≥下限"），故从 config / dataclass / 文档中移除该失效参数（非实现错误约束）。
- `backtest_pipeline.run_backtest()`：该函数无人调用且用旧重构基准，与新流程不一致，已删除（两个 demo 已覆盖完整流程）。
- 文档漂移：`__init__` 指向已删 manual.md 已改为操作指南；`index_enhance_method` 标题改为中证1000；`design.md` 模块结构 / 数据流 / 关键决策更新到真实实现。
- 测试 + CI：新增 `tests/`（指数加载 / 风格上限解析 / 配置 / 优化器约束含停牌为0、单因子约束生效，16 项）+ GitHub Actions（pytest 强制 + ruff advisory）。

**评估后未改（附理由）**
- 求解后归一化越界：clip 仅去除约 1e-9 量级负值，再归一化的越界在求解器容差量级，实测无意义，不改以免无谓改动回测结果。
- 流动性"硬约束"：实现确为软惩罚，已改 `design.md` 措辞与实现对齐（未新增 ADV 硬约束这一新功能）。

**经数据核实后撤销的疑虑（codex/初判有误）**
- 停牌"估值=0 / NAV 失真 / 卖单 p<=0 早退被丢弃"：**不成立**。实测行情在停牌日对价格字段（adj_close / adj_vwap）做前值填充（仅 volume/amount=0，2025 年 3441 条停牌记录 0% null / 0% zero）。故停牌持仓按最近价估值（NAV 不失真），停牌卖单 `exec_p>0` 正确进入延期队列、复牌后成交。当前"优化器目标 w=0 + 引擎延期卖出"自洽且贴近现实，无需改动。
- `realistic_engine.py:188` 的 `if p<=0: continue` 仅对价格真缺失（NaN）的边缘情况（如退市）才会早退；停牌不触发。属低优先级代码健壮性项，非回测口径问题。
