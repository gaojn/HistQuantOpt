## P2：停牌/跌停处理仍需明确业务口径和测试覆盖

位置：
- `portfolio_optimizer/optimizer/alpha_max.py:146`
- `portfolio_optimizer/optimizer/alpha_max.py:167`
- `portfolio_optimizer/optimizer/index_enhance.py:129`
- `portfolio_optimizer/optimizer/index_enhance.py:149`
- `portfolio_optimizer/backtest/engine.py:244`
- `portfolio_optimizer/backtest/engine.py:297`

当前优化器：
- 停牌、次新：`w_i = 0`
- 涨停：`w_i <= w_prev_i`
- 跌停：`w_i >= w_prev_i`

当前回测器：
- 停牌不可买卖。
- 跌停不可卖，进延期队列。
- 涨停不可买，资金留现金。

这套设计可以解释为“优化器表达目标持仓，回测器处理实际成交”。但它会带来一个重要结果：停牌股票在目标权重里被打到 0，实际持仓却因为无法成交继续留在组合里，目标组合和实际组合会长期偏离。

建议：
- 明确产品口径：优化器到底输出“目标权重”还是“可交易后预期持仓”。
- 若输出目标权重，报告和测试要覆盖“停牌目标卖出但实际延期”的偏离。
- 若输出可交易后持仓，停牌应固定 `w_i = w_prev_i`，跌停也应至少不低于 `w_prev_i`。

当前新增测试只断言停牌权重为 0，没有覆盖回测延期执行后的实际持仓/NAV 逻辑。
